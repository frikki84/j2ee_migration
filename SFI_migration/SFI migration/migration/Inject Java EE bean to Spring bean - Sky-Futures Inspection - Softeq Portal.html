Inject Java EE bean to Spring bean

Skip to end of metadata
  • Created by Alexander Milosh, last modified on Jun 23, 2020
Go to start of metadata

You can inject Java EE bean which annotated with @Stateless only. During application building java bean is created and put to Java Container. This bean has a JNDI name. Using this name you can get access to bean from outer resources.

To know JNDI name for particular bean run application and see log (e.g. for TableResultBoundary):

run log
?
INFO  [org.jboss.as.ejb3.deployment.processors.EjbJndiBindingsDeploymentUnitProcessor] (MSC service thread 1-6) JNDI bindings for session bean named TableResultsBoundary in deployment unit deployment "employee.war" are as follows:
 
    java:global/employee/TableResultsBoundary!com.softeq.project.fwk.component.table.boundary.TableResultsBoundary
    java:app/employee/TableResultsBoundary!com.softeq.project.fwk.component.table.boundary.TableResultsBoundary
    java:module/TableResultsBoundary!com.softeq.project.fwk.component.table.boundary.TableResultsBoundary
    java:global/employee/TableResultsBoundary
    java:app/employee/TableResultsBoundary
    java:module/TableResultsBoundary

To inject Java EE bean to Spring bean you should use annotation @Resoiurce and specify JNDI name:

TableControllerSpring.java
?
import javax.annotation.Resource;
 
@Controller
public class TableControllerSpring {
    @Resource(mappedName = "java:global/employee/TableResultsBoundary!com.softeq.project.fwk.component.table.boundary.TableResultsBoundary")
    private TableResultsBoundary tableBoundary;
}
