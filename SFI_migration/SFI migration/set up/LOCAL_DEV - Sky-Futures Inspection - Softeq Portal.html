Local developer environment configuration

Setup

  • Install Java 8
  • Install the latest maven
  • Setup your local dev environment according to Softeq conventions
  • Install Git
  • Checkout the current code-base from stash
  • Install MySQL with default user\password pair root\root

  • Execute script ../misc/sql/create_db_and_users.sql for create DB and SF user

    ?
    mysql -u root -p < misc\sql\create_db_and_users.sql

    OR

    Open SQL client (MySQL Workbench, IntelliJ IDEA Database tab or other). Run queries from file: %SFI_FOLDER%/misc/sql/create_inspection_template_example.sql

  • Download pre-configured WildFly

    ?
    \\cls-fs.softeq.com\departments\Enterprise web\Java Unit\projects\SF\skyfuture_wildfly-9.0.1.Final.zip

    Unzip it.

  • Set `JBOSS_HOME` env variable to pre-configured Wildfly

  • Create your own IDEA Run configuration for WildFly

    In the Run/Debug configuration for your server in the tab Startup/Connection you have the ability to set Startup script: on the end of line there is checkbox Use default.
    Please unselect it and paste on the end of the input `--server-config=standalone-full.xml`
    Look out for different startup scripts for "Run" and "Debug" in Intellij IDEA.
    If you don't uncheck "Use default" in both of them then you can end up with two different profiles on "Run" and "Debug".
    It is easy to forget and annoying to figure it out.

  • Add exploded web modules to your server:

    • employee

    • customer

    • mpc - only if you are developing mpc, otherwise do not add it. By default your local server is configured to connect to shared softeq server where mpc already deployed.

  • Switch to branch 'develop'. (In modal window 'These modules have been removed from Maven structure: 'entity'' click on 'Yes')

  • Adapt to use shared queue

    In local dev in order to get from shared softeq server only callback messages produced by you - we need to filter messages by some unique identify.

    Every developer have to copy: 

    1. "jboss-ejb3.xml" from "\misc\local_dev\employee" into "\web-app\employee\src\main\webapp\WEB-INF\", specify your MAC

    2. "jboss-ejb3.xml" from "\misc\local_dev\customer" into "\web-app\customer\src\main\webapp\WEB-INF\", specify your MAC

      NOTE - find the class com.softeq.project.fwk.util.MacUtils, it helps you check your MAC-address

       

  • Issue: java8 + VRaptor:

    Settings/Java Compiler >> additional command line parameters : -parameters

  • Add AWS credentials to your HOME directory. The file should be placed at ~/.aws/credentials

    Example of <home directory>/.aws/credentials
    ?
    [default]
    aws_access_key_id = AKIAIAXS5JIC667QPHMA
    aws_secret_access_key = 0jxrE3jDVuiMZVo2h4BLLpNv+x3smtB/3t1wvDk+
  • Run application server and enjoy!
  • Use the following credentials: credentials.
  • Set IntelliJ IDEA code style: SkyFuturesStyle.xml

Developing MPC

In order to run MPC on your local server - you need in definition of pooled connection factory named "hornet-ra-mpc" change connector`s name from "netty-mpc" to "netty-mpc-dev-connector" in your $JBOSS_HOME/standalone/configuration/standalone-full.xml

?
<pooled-connection-factory name="hornetq-ra-mpc">
    ***
    <connectors>                         
        <!-- <connector-ref connector-name="netty-mpc"/> -->
        <connector-ref connector-name="netty-mpc-dev-connector"/>
    </connectors>
    ***
</pooled-connection-factory>

Media libs installation to the local developer environment

In order to run MPC on your local server - you need to setup AVConv and GM libs on your local instance, too.

  • If your target OS is Unix -  perform installation steps from here
  • If your target OS is Windows:
    - download GraphicsMagic for Windows(choose Q16 version)
    - download \\cls-fs.softeq.com\departments\Enterprise web\Java Unit\projects\SF\avconv-bin.rar and unzip it

  • Override system variables:
    gm.path and avconv.path by setting pathes to the installed libraries
    gm.temp.dir and avconv.temp.dir by setting pathes to the temporary directories for the installed libraries

    If your custom path contains spaces, update part  <system-properties> in standalone-full.xml like that:

    <property name="gm.path" value="C://Program Files//GraphicsMagick-1.3.21-Q16//gm"/>

    Not all places properly work with spaces. Please avoid using spaces in temp dirs to exclude hard-to-find bugs on local mpc.

    The config in your standalone-full.xml example

    <system-properties>
    <property name="appEnv" value="LOCAL_DEV"/>
    <property name="gm.path" value="C:\Program Files\GraphicsMagick-1.3.26-Q16\gm.exe"/>
    <property name="avconv.path" value="D:\avconv\"/>
    <property name="gm.temp.dir" value="D:\TEMP\GraphicsMagick_TEMP"/>
    <property name="avconv.temp.dir" value="D:\TEMP\avconv_TEMP"/>
    </system-properties>

  • Don't forget to add AWS credentials to your HOME directory. The file should be placed at ~/.aws/credentials

    Example of <home directory>/.aws/credentials
    ?
    [default]
    aws_access_key_id = AKIAIAXS5JIC667QPHMA
    aws_secret_access_key = 0jxrE3jDVuiMZVo2h4BLLpNv+x3smtB/3t1wvDk+


Local WildFly manual configuration

This section describes how manually configure WildFly for local development: all steps to produce pre-configured zip.

Common configuration

perform common steps

Specific configuration

$JBOSS_HOME/standalone/configuration/standalone-full.xml

Allow switching to MPC local development w/o code changes

Add additional connector and acceptor
?
<subsystem xmlns="urn:jboss:domain:messaging:3.0">
    ***
    <connectors>
        <netty-connector name="netty-mpc-dev-connector" socket-binding="mpc-jms-broker-dev-connector"/>
    </connectors>
    <acceptors>
        <netty-acceptor name="netty-mpc" socket-binding="mpc-jms-broker-dev-acceptor"/>
    </acceptors>
    ***
</subsystem>
Add additional socket bindings
?
<socket-binding-group name="standard-sockets" default-interface="public" port-offset="${jboss.socket.binding.port-offset:0}">
    <socket-binding name="mpc-jms-broker-dev-acceptor" port="${mpc.broker.port:5445}"/>
    <outbound-socket-binding name="mpc-jms-broker-dev-connector">
        <remote-destination host="${mpc.host:127.0.0.1}" port="${mpc.broker.port:5445}"/>
    </outbound-socket-binding>
</socket-binding-group>
